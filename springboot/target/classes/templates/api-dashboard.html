API Dashboard Documentation

Comprehensive API endpoints for user management and system monitoring. All endpoints require ADMIN role authentication.

Dashboard Overview

GET /dashboard/overview

Description: Get main dashboard statistics and metrics

Response: User stats, session stats, operation stats

{ "success": true, "data": { "users": { "total": 25, "active": 20, "verified": 18 }, "sessions": { "total": 150, "completed": 120, "failed": 10 }, "operations": { "total": 500, "successful": 450, "successRate": "90.0%" } } }
GET /dashboard/statistics

Description: Get detailed system statistics

Response: Comprehensive system metrics including TTN and ANCE operations

User Management

GET /dashboard/users

Description: List all users with filtering and pagination

Parameters:

  • page (int): Page number (default: 0)
  • size (int): Page size (default: 20)
  • sortBy (string): Sort field (default: createdAt)
  • sortDir (string): Sort direction (default: desc)
  • search (string): Search term (optional)
  • role (string): Filter by role (optional)
  • isActive (boolean): Filter by active status (optional)
GET /dashboard/users/{id}

Description: Get detailed user information

Response: Complete user profile with statistics (credentials status only, not actual values)

GET /dashboard/users/{id}/sessions

Description: Get user's workflow sessions

Parameters: page, size

Response: Paginated list of user's workflow sessions

System Monitoring

GET /api/v1/dashboard/activity

Description: Get recent system activity

Parameters:

  • limit (int): Number of recent activities (default: 50)

Response: Recent operations and workflows

Authentication

Authentication Required

All dashboard API endpoints require:

  • JWT Token: Include in Authorization header as Bearer {token}
  • ADMIN Role: User must have ADMIN role

Example:

Authorization: Bearer eyJhbGciOiJIUzUxMiJ9...

Usage Examples

JavaScript/Fetch Example:

// Get dashboard overview fetch('/api/v1/dashboard/overview', { headers: { 'Authorization': 'Bearer ' + token, 'Content-Type': 'application/json' } }) .then(response => response.json()) .then(data => { if (data.success) { console.log('Dashboard data:', data.data); } }); // Get users with filtering fetch('/api/v1/dashboard/users?search=john&role=USER&page=0&size=10', { headers: { 'Authorization': 'Bearer ' + token, 'Content-Type': 'application/json' } }) .then(response => response.json()) .then(data => { console.log('Users:', data.data); console.log('Pagination:', data.pagination); });

cURL Example:

# Get dashboard overview curl -X GET "http://localhost:8080/api/v1/dashboard/overview" \ -H "Authorization: Bearer YOUR_JWT_TOKEN" \ -H "Content-Type: application/json" # Get users with search curl -X GET "http://localhost:8080/api/v1/dashboard/users?search=admin&page=0&size=5" \ -H "Authorization: Bearer YOUR_JWT_TOKEN" \ -H "Content-Type: application/json"
